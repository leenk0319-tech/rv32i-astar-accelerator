# 2026-06-25 개발일지 - RV32I Pipeline Branch Flush 검증

## 프로젝트 개요

- 프로젝트명: RV32I A* Accelerator SoC
- 목표: RV32I 기반 CPU를 직접 구현하고, 이후 MMIO bus와 A* pathfinding accelerator를 통합하는 FPGA SoC 구현
- 타겟 보드: Terasic DE2-115 / Intel Cyclone IV
- 언어: Verilog HDL
- 현재 단계: 5-stage pipeline CPU의 branch 처리와 flush 검증

## 오늘 구현한 내용

오늘은 단순 ALU/레지스터 writeback 검증에서 한 단계 넘어가서, pipeline CPU에서 branch가 taken 되었을 때 잘못 들어온 instruction을 제거하는 flush 흐름을 구현하고 Questa로 검증했다.

핵심 변경은 branch 판단을 EX stage에서 수행하고, branch taken이면 IF/ID와 ID/EX pipeline register를 NOP/bubble로 만드는 구조이다.

```verilog
wire branch_taken;
assign branch_taken = E_branch && E_branch_condition;
```

이 신호는 다음 세 곳에서 사용된다.

1. PC 선택: taken이면 branch target address로 이동
2. IF/ID flush: 이미 fetch된 wrong-path instruction을 NOP으로 변경
3. ID/EX flush: decode된 wrong-path instruction의 control signal을 0으로 만들어 bubble 처리

## 설계 결정

### Branch 판단 위치

현재 구현에서는 branch condition을 EX stage의 ALU 쪽에서 계산한다. ID stage에서 branch를 판단하면 penalty는 줄일 수 있지만, forwarding/hazard 처리가 더 복잡해진다. 아직 forwarding과 stall unit이 없기 때문에, 지금 단계에서는 EX stage 판단이 더 안전하고 구현 난이도도 적절하다.

### Flush 범위

Branch가 EX stage에서 taken으로 판정되는 순간, 그보다 뒤에 있는 instruction은 IF/ID와 ID/EX에 들어가 있다. 따라서 flush 대상은 IF/ID와 ID/EX이다.

- IF/ID: instruction을 `32'h00000013`으로 변경
- ID/EX: control signal을 전부 0으로 변경
- EX/MEM, MEM/WB: branch 자신 또는 그 이전 instruction이므로 flush하지 않음

## NOP 처리

RISC-V에서 NOP은 아래 instruction이다.

```verilog
32'h00000013  // addi x0, x0, 0
```

IF/ID register에서는 reset 또는 flush 시 instruction field에 이 값을 넣었다. ID/EX register에서는 instruction 자체보다 control signal을 모두 0으로 만드는 방식이 중요하다. 이렇게 하면 register write, memory write 같은 부작용이 발생하지 않는다.

## 추가한 검증 파일

오늘 추가한 검증 파일은 다음과 같다.

| 파일 | 역할 |
|---|---|
| `Risc_V/6_24_25/program.hex` | branch taken/not-taken 검증용 instruction memory image |
| `Risc_V/6_24_25/tb_branch_nohazard.v` | branch flush 검증용 Questa testbench |

`program.hex`에는 BEQ, BNE, BLT, BGE가 포함되어 있다. 아직 forwarding/stall이 없으므로 RAW hazard를 피하기 위해 dependent instruction 사이에 NOP을 삽입했다.

## Questa 검증 결과

Questa FSE에서 다음 순서로 검증했다.

```tcl
cd D:/Programs/vscode_workspace/Soc_Project/Risc_V/6_24_25
vlib work
vlog -sv *.v
vsim tb_branch_nohazard
run -all
```

실행 결과:

```text
PASS: branch taken/not-taken flush test without RAW hazards
```

검증된 내용은 다음과 같다.

- BEQ taken 시 wrong-path instruction이 flush됨
- BNE not taken 시 fall-through instruction이 정상 실행됨
- BLT taken 시 wrong-path instruction이 flush됨
- BGE taken 시 wrong-path instruction이 flush됨

## 현재 한계

현재 CPU는 아직 완성된 RV32I pipeline CPU가 아니다.

- Forwarding unit 없음
- Stall/hazard detection 없음
- Load-use hazard 처리 없음
- JAL/JALR datapath는 아직 완성 전
- MMIO bus 및 A* accelerator는 아직 통합 전

따라서 현재 테스트는 hazard를 의도적으로 피한 instruction sequence로 구성했다.
